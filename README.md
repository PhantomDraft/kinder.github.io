# Скрипт генерации конфига на контрольные листы по математике для первого-второго класса

**Адаптировано под**: https://phantom-draft.com/kinder.github.io/poll.html

## Создание JSON-файла

Файл JSON можно создать, например, в текстовом редакторе «Блокнот». Для этого:

1. Откройте новый файл.
2. Вставьте в него код конфигурации.
3. Сохраните файл с расширением `.json` (вместо `.txt`).

**Примечание 1:** Данный текст можно использовать лично или передать AI в качестве технического задания (копировать от блока «Сценарий» до «Задача для AI» включительно). В последнем случае рекомендуется самостоятельно вычитать и при необходимости скорректировать его.

**Примечание 2:** В Windows по умолчанию скрыты расширения файлов, из-за чего при переименовании файл может не изменить реальный формат. Чтобы включить отображение расширений, выполните следующие шаги:

- Откройте Проводник Windows
- Перейдите на вкладку **Вид**
- Установите галочку **Расширения имен файлов**
- Готово — теперь вы видите и можете менять расширения файлов

## Структура задания

Каждый объект в массивах `standard` и `advanced` описывает отдельное упражнение. Обязательное поле — `text`. В зависимости от типа задания указываются:

- `dataAnswer` — строка с правильным ответом. Используется для задач с вводом текста;
- `options` — массив вариантов для выбора. Каждый элемент может иметь вид `{ "text": "7", "correct": true }`;
- `hint` — необязательная подсказка, отображается под формулировкой.

Как минимум одно из полей `dataAnswer` или `options` должно присутствовать.

## Сценарий

### Общие инструкции
- Создать два листа самостоятельной работы для первоклассников по арифметике.
- Числовые значения, условия и ситуации в заданиях должны генерироваться случайным образом (уникализироваться) для каждого нового варианта.
- Задания должны быть сформулированы так, чтобы ученик мог решить их самостоятельно, без дополнительных подсказок.

### Лист 1: Стандартные задания
В этой части задания формулируются по образцу стандартных упражнений, примеры заданий:

- Текст: «Первое слагаемое **X**, второе слагаемое **Y**. Запишите сумму.»
- Цель: Отработка понятия «слогаемое» и «сумма».

- Текст: «Запишите число, в котором **A** десятков и **B** единиц.»
- Цель: Работа с разрядностью (однозначное/двузначное число).

- Текст: «Число **C** увеличьте на **D**.»
- Цель: Отработка понятия «последующее число».

- Текст: «Уменьшаемое **E**, вычитаемое **F**. Запишите разность.»
- Цель: Закрепление терминов «уменьшаемое», «вычитаемое», «разность».

- Текст: «Запишите соседей числа **G**.»
- Цель: Тренировка навыка нахождения «соседнего числа».

6. **Наименьшее двузначное число**
- Текст: «Наименьшее двузначное число.»
- Цель: Определение минимального числа заданного разряда.

- Текст: «К разности чисел **H** и **I** прибавьте **J**.»
- Цель: Совмещённое применение операций вычитания и сложения.

8. **Нахождение недостающего слагаемого**
- Текст: «Какое слагаемое нужно прибавить к **K**, чтобы получить **L**?»
- Цель: Поиск недостающего числа при сложении (решение уравнения).

9. **Задача со временем**
- Текст: «Если игра начнётся через **M** секунд, а текущее время **T**, во сколько стартует игра?»
- Цель: Применение арифметики для вычисления времени.

*Примечание:* Все обозначения (**X**, **Y**, **A**, **B**, …, **T**) должны заменяться на случайные уникальные значения.

### Лист 2: Задания на опережение
В этой части задания разрабатываются на основе нашего шаблона с дополнительными вопросами для закрепления материала и углубления в терминологию. Примеры заданий:

1. **Определение следующего числа**
 - Пример: «Заполните пропуск: **N**, ___, **P**. Как называется число, которое идёт сразу после **N**?»
- Цель: Закрепление понятия «последующее число».

2. **Определение предыдущего числа**
 - Пример: «Заполните пропуск: ___, **Q**, **R**. Какое число находится непосредственно перед **Q**?»
- Цель: Закрепление понятия «предыдущее число».

3. **Нахождение соседа в последовательности**
- Пример: «В последовательности: **S**, ___, **U** – какое число должно стоять между **S** и **U**?»
- Цель: Отработка понятия «соседнее число».

4. **Наименьшее трёхзначное число**
- Текст: «Наименьшее двузначное число.»
- Цель: Определение минимального числа заданного разряда.

5. **Арифметическая операция сложения**
 - Пример: «Выполните: **V** + **W** = ?»
- Цель: Отработка понятия сложения и идентификация слагаемых и суммы.

6. **Заполнение пропусков в числовом ряду**
- Пример: «Последовательность: **X1**, ___, **Z1**, ___, **B1**. Какое число идёт после **X1** при прибавлении **Y1**?»
- Цель: Тренировка нахождения следующего числа по правилу.

7. **Сравнение чисел**
- Пример: «Даны числа: **C1**, **C2**, **C3**. Какое из них наименьшее, а какое – наибольшее?»
- Цель: Упражнение на сравнение числовых величин.

8. **Ситуационная задача с тетрадями**
- Пример: «Купили **D1** тетрадей в клетку, а в линейку – на 1 больше. Сколько тетрадей купили всего?»
- Цель: Применение арифметики в реальных ситуациях.

9. **Выбор правильного арифметического знака**
- Пример: «**E1** __ **E2** = **E3**. Какой знак (+ или –) нужно поставить, чтобы равенство было верным?»
- Цель: Отработка выбора арифметического знака.

10. **Определение чётности числа**
- Пример: «Определи, какое из чисел **F1**, **F2**, **F3** является чётным. Какой признак позволяет отнести число к чётным?»
- Цель: Закрепление понятий чётности и нечётности.

11. **Счёт шагами (с пропусками)**
 - Пример: «Заполните пропуски: **G1**, ___, **G3**, ___, **G5**, если каждое число увеличивается на **H1**.»
- Цель: Тренировка счёта с заданным шагом.

12. **Нахождение лишнего числа**
 - Пример: «В последовательности: **J1**, **J2**, **J3**, **J4**, **J5** одно число выбивается. Найдите лишнее и обоснуйте свой выбор.»
- Цель: Поиск закономерностей и анализ последовательности.

13. **Сравнение фигур по количеству сторон**
 - Пример: «Сравните фигуры: треугольник (3 стороны) и прямоугольник (4 стороны). Заполните пробел знаком сравнения, чтобы указать, у какой фигуры больше сторон.»
- Цель: Отработка сравнения количественных характеристик.

14. **Задача «Больше, чем на…»**
- Пример: «У **K1** на **K2** предмета больше, чем у **L1**. Если у **L1** всего **K3** предметов, сколько предметов у **K1**? Как записать эту ситуацию в виде арифметического выражения?»
- Цель: Закрепление понятия «на больше» через составление арифметического выражения.

*Примечание:* Все обозначения (**N**, **P**, **Q**, **R**, **S**, **U**, **V**, **W**, **X1**, **Y1**, **Z1**, **B1**, **C1**, **C2**, **C3**, **D1**, **E1**, **E2**, **E3**, **F1**, **F2**, **F3**, **G1**, **G3**, **G5**, **H1**, **J1**–**J5**, **K1**, **K2**, **L1**) должны заменяться на случайные уникальные значения при генерации.

## Задача для AI

Создать два листа самостоятельной работы по выше указанным инструкциям из раздела 'Сценарий'.
В ответ должно быть выдано два листа с заданиями, уникальными числовыми значениями и условиями, соответствующими приведённым инструкциям.
**Формат ответа пользователю** — JSON с двумя основными разделами:

* standard – Лист 1: стандартные задания.
* advanced – Лист 2: задания на опережение.

Каждый объект в массивах представляет отдельное задание и содержит по крайней мере два ключа:

* text: строка с формулировкой задания. В тексте задания используются числовые значения и обозначения (например, X, Y, A, B, ...), которые при генерации нового варианта заменяются на случайные уникальные значения.
* dataAnswer: строка с корректным ответом для проверки.

### Пример корректного выходного конфига

```json
{
  "standard": [
    { "text": "Первое слагаемое 8, второе слагаемое 5. Запишите сумму.", "dataAnswer": "13" }
  ],
  "advanced": [
    { "text": "Заполните пропуск: 4, ___, 6. Как называется число, которое идёт сразу после 4?", "dataAnswer": "5" }
  ]
}
```
